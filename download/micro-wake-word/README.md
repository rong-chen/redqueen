# microWakeWord 语音唤醒词库 (中文说明与工具指南)

`microWakeWord` 是一个开源的唤醒词检测和训练库，专为在低功耗嵌入式设备（如 ESP32、ESP32-S3）上运行而设计。它生成的模型格式为标准的 TensorFlow Lite for Microcontrollers (`.tflite`)，在真实使用场景下具有极低的误唤醒率和漏唤醒率。

## microWakeWord 唤醒检测原理解析

### 1. 两个阶段的检测流程
*   **第一阶段（音频前端预处理）**：
    从麦克风捕获的 16 kHz 单声道原始声音，每隔 10ms 提取一个 30ms 长度的音频窗。预处理器（利用 Google 官方的 `micro_speech` 音频前端）将时域信号转化为 40 个频带的 Log-Mel 能量值，并内置了噪声抑制和自动增益控制（AGC）。这相当于生成声音的“指纹特征图”。
*   **第二阶段（神经网络推理）**：
    流式推理神经网络（如 MixConv 混合深度可分离卷积网络）每 30ms 执行一次计算。它通过滑窗的形式输入最新的特征切片，并输出当前音频中包含唤醒词的概率（0.0 到 1.0）。如果在连续多个时间窗内输出概率均维持在较高阈值之上，则判定“设备被成功唤醒”。

### 2. 训练机制与数据增强
*   **数据增强**：训练时会采用 SpecAugment 技术，随机遮蔽一部分时间和频率特征，增强模型在真实噪音环境下的鲁棒性。
*   **环境噪声比对**：测试和验证集除了包含正负人声样本外，还引入了专门的 `ambient` 环境数据集（包含音乐、家电噪音、日常闲聊等），以极力压低设备在日常背景音中的“误唤醒率 (FAR)”。
*   **量化转换**：为了能在 ESP32-S3 等微控制器上流畅运行，导出的 streaming 模型会做 INT8 量化，这能大幅降低内存消耗（通常仅需 150KB-300KB RAM）并最大化推理速度。
