# RedQueen System (红皇后智能语音中枢)

本项目是一个全栈式本地智能语音交互控制中心（RedQueen）。包含 Vue 3 管理后台前端、Golang 高性能 WebSocket/API 后端，以及一套专门配套使用的离线唤醒词训练工具集（microWakeWord 炼丹炉）。

## 系统核心架构

1. **管理前端 (`admin-ui/`)**: 
   - 提供语音交互日志看板、MCP 服务接入控制。
   - **双重声纹管理 (`/voiceprint-enroll`)**: 用于录入主人的声纹，实现极高安全性的语音验证门禁。
2. **中枢后端 (`controllers/`, `services/`, `models/`)**: 
   - Golang 高并发 WebSocket 服务器，负责接收前端语音流。
   - 集成 `WeSpeaker` (基于 ONNX) 声纹识别，严格把关每一个语音请求，仅允许注册用户（如：主人）控制设备。

## 🛠️ 硬件接入方案推荐：ESP32 智能语音端

为了实现最佳的远场唤醒与本地消噪体验，推荐结合开源硬件项目使用：

### 推荐硬件固件：`xiaozhi-esp32` (小智)
本系统完美支持接收来自第三方硬件的 WebSocket 音频流。推荐使用国内火热的开源固件 `xiaozhi-esp32`：

1. **获取硬件**：使用带有麦克风和扬声器模块的 ESP32-S3 开发板。
2. **刷入固件**：使用小智提供的 Web Flasher 一键刷机。
3. **配置唤醒词**：在小智的配置后台中，选择乐鑫官方高度优化的极速唤醒词 **“嗨，小冰”**。该模型基于乐鑫 ESP-SR 框架，自带硬件级回声消除（AEC）与环境降噪（NS），抗噪能力极强且无需任何额外训练。
4. **连接红皇后**：将固件后台的 **WebSocket 服务器地址** 更改为本系统 Golang 后端的地址：
   `ws://<您的服务器IP>:8080/ws/voice`

### 军工级双重防御机制
当您的硬件端喊出“嗨，小冰”唤醒后：
1. **硬件极速唤醒**：ESP32 本地的 ESP-SR 芯片级算法完成唤醒和降噪，并将纯净的音频流推送至后端。
2. **声纹看门狗拦截**：音频流到达 Golang 后端后，底层的 `WeSpeaker` 声纹大模型会进行二次精准校验。如果提取到的声纹特征与已注册的主人声纹不符，无论说了什么指令都将直接被切断通信，防止旁人恶意唤醒。
